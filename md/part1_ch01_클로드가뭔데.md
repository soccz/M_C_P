# Ch.1 클로드가 뭔데?

> **이 챕터를 마치면**: 클로드가 단순한 챗봇이 아니라 4개의 얼굴을 가진 작업 시스템이라는 걸 이해하고, 자신의 상황에 맞는 경로를 고를 수 있다.

---

## 배경: AI 챗봇이라고 부르지 마세요

"클로드는 AI 챗봇이다"라는 말을 자주 듣는다. 틀린 말은 아니지만, 이렇게 이해하면 클로드의 10%만 쓰게 된다.

챗봇이라고 부르는 순간, 우리는 "질문하면 답하는 것"만 기대한다. 마치 검색엔진에 질문을 입력하는 것처럼. 하지만 클로드는 질문에 답하는 것 이상을 할 수 있다.

- 파일을 읽고 수정할 수 있다
- 코드를 작성하고 테스트를 돌릴 수 있다
- 외부 서비스(Slack, GitHub, Notion 등)에 접속할 수 있다
- 정해진 시간에 자동으로 작업을 수행할 수 있다
- 여러 작업을 동시에 나눠서 처리할 수 있다

이건 챗봇이 아니다. **작업 파트너**다.

> **💭 생각의 흐름**
>
> **문제** — 처음에 나도 클로드를 "똑똑한 검색엔진"으로만 썼다
> **질문** — "왜 사람들은 Claude Code가 다르다고 하는 거지?"
> **발견** — 같은 모델인데 어디서 쓰느냐에 따라 완전히 다른 결과가 나온다
> **결과** — 4가지 경로를 이해하는 것이 클로드 활용의 첫 단계였다

---

## 클로드의 4가지 얼굴

같은 클로드 모델이지만, **어디서 만나느냐**에 따라 경험이 완전히 달라진다. Anthropic은 크게 4가지 경로를 제공한다.

### 1. Chat — 질문하고 답을 받는 곳

**claude.ai**에서 대화창을 열면 이게 Chat이다.

**적합한 일:**
- 아이디어 발상과 브레인스토밍
- 빠른 요약과 번역
- 간단한 분석과 의견 요청
- 코드 질문 (짧은 것)

**이런 느낌:** 카페에서 똑똑한 친구에게 물어보는 것. 가볍고 빠르다.

**제약:** 매 대화가 독립적이다. 어제 대화 맥락을 오늘 기억하지 못한다 (Projects를 쓰지 않으면). 파일을 직접 수정하거나 코드를 실행할 수는 없다.

**직무별 활용:**
- 창업가 / 1인 사업자: 생각 정리, 배경 지식 누적
- PM / 기획: 질문 정리, 요구사항 초안
- 마케팅: 빠른 요약, 카피 아이디어
- 개발자: 빠른 아이디어와 코드 생성
- 분석가 / 연구자: 데이터 해석 질문, 논문 요약

---

### 2. Projects — 같은 배경 자료를 참고하고 가는 곳

Chat과 비슷하지만, **프로젝트 단위로 배경 자료를 누적**할 수 있다.

**적합한 일:**
- 한 번 설명한 배경을 다시 말하고 싶지 않을 때
- 특정 프로젝트의 파일과 컨텍스트를 묶어 관리할 때
- 회의록, 리서치, 브리프를 쌓아가면서 참조할 때

**이런 느낌:** 사무실 책상 위에 관련 자료를 펼쳐놓고 일하는 것.

**Chat과의 차이:**
- Chat: 매번 0부터 설명해야 한다
- Projects: "지난번 회의록 참고해서" 같은 맥락이 가능하다

**직무별 활용:**
- 창업가: 배경 자료를 계속 쌓고 가는 것
- PM: 기능 배경 유지, 스펙 문서 누적
- 마케팅: 반복 배경 유지, 캠페인 히스토리
- 개발자: 프로젝트 아키텍처 문서, 코드 컨벤션 참조
- 분석가 / 연구자: 논문·데이터 배경 누적, 분석 히스토리

---

### 3. Cowork — 문서와 브리프 결과물을 맡기는 곳

여기서부터 성격이 달라진다. Chat이나 Projects는 **내가 화면 앞에 있어야** 한다. Cowork은 **맡기고 자리를 비울 수 있다**.

**적합한 일:**
- 주간 경쟁사 브리프 작성
- 회의 정리와 후속 조치
- 리서치 보고서
- 자료 분류와 저장

**이런 느낌:** 유능한 비서에게 "이거 정리해놓고 내일 아침까지 보내줘"라고 하는 것.

**핵심 차이점:**
- 비동기 위임이 가능하다 — 내가 없어도 일한다
- 결과물이 파일로 남는다 — 채팅 로그가 아니라 정리된 문서
- Scheduled Tasks로 반복 자동화가 가능하다

**Cowork 폴더 구조:**
```
CLAUDE-COWORK/
├── ABOUT-ME/          ← 나는 누구인가
│   ├── about-me.md
│   └── working-rules.md
├── PROJECTS/          ← 이번 작업에 필요한 것
│   └── client-a/
│       └── brief.md
├── TEMPLATES/         ← 출력 형식 고정
│   └── weekly-brief-template.md
└── CLAUDE-OUTPUTS/    ← 결과물 저장
```

**직무별 활용:**
- 창업가: 시장 브리프와 조건, MVP 수행과 검증
- PM: PRD 초안, 회의 정리 자동화
- 마케팅: 주간 브리프, 콘텐츠 아이디어, 일정 기준 반복
- 개발자: 코드 리뷰 요약, 기술 조사 보고서
- 세일즈: 리서치, 제안서, 비교표, 리포트
- 분석가 / 연구자: 데이터 분석 보고서, 트렌드 모니터링
- 백오피스/운영: 문서 정리와 자동화 중심

> **💭 생각의 흐름**
>
> **문제** — 매주 경쟁사 동향을 정리하는 게 귀찮다
> **질문** — "이걸 클로드가 알아서 해줄 수 없나?"
> **발견** — Cowork의 Scheduled Tasks로 매주 월요일 자동 실행 가능
> **결과** — "월요일 출근하면 브리프가 와 있다" 경험을 처음 함

---

### 4. Claude Code — 코드를 짜고 테스트하고 검증하는 곳

**터미널**(까만 화면)에서 `claude`를 입력하면 시작된다.

**적합한 일:**
- 코드 작성, 수정, 리팩터링
- 버그 수정과 테스트
- 파일 시스템 탐색과 수정
- Git 작업 (커밋, 브랜치, PR)
- MCP로 외부 도구 연결
- 복잡한 자동화 파이프라인 구축

**이런 느낌:** 옆에 앉아서 같이 코딩하는 시니어 개발자.

**Chat과의 근본적 차이:**
- Chat은 코드를 **보여준다** (복사해서 붙여넣어야 한다)
- Claude Code는 코드를 **직접 실행한다** (파일을 만들고, 테스트를 돌리고, Git에 커밋한다)

**핵심 키워드:**
- CLAUDE.md, Skills, Hooks, MCP — 이 모든 것을 설정하고 활용할 수 있는 곳
- worktree — 병렬 작업 폴더
- SubAgent — 보조 에이전트로 작업 분담

**직무별 활용:**
- 개발자: 코드 수정, 테스트, 배포, 리팩터링의 **주 작업 도구**
- 창업가: MVP 프로토타입 빌드, 자동화 파이프라인 구축
- PM: 데이터 추출 스크립트, 배포 자동화 연동
- 마케터: 콘텐츠 파이프라인 자동화, 발행 스크립트 실행
- 분석가: 데이터 처리 스크립트, 시각화 자동화
- 비개발자: 터미널이 낯설다면 Cowork부터 시작해도 된다

## 12 Surface 지도 🆕 — 같은 세션이 12곳을 타고 흐른다

2026년 들어 Anthropic은 Claude Code를 **단일 엔진, 다중 Surface** 구조로 재편했다. 같은 CLAUDE.md, 같은 MCP, 같은 Skills가 12개 입구 중 어디서 열어도 동일하게 적용된다. 4경로는 사용 목적의 분류이고, 12 Surface는 **실제 접속하는 입구**다.

### 12 Surface 한눈에 보기

| 분류 | Surface | 어디서 열리나 | 강점 |
|---|---|---|---|
| **터미널 계열** | Claude Code CLI | 내 컴퓨터 터미널 | 최강 능력치 (MCP/Skills/Hooks 전부) |
| **IDE 계열** | VS Code 확장 | VS Code 사이드바 | 인라인 diff, @-mention |
| | JetBrains 플러그인 | IntelliJ/PyCharm/WebStorm | 네이티브 IDE 통합 |
| | Cursor 확장 | Cursor 사이드바 | VS Code 확장 호환 |
| **앱 계열** | Desktop 앱 | Mac/Windows 네이티브 앱 | 시각 diff, 다중 세션, `/desktop` 핸드오프 |
| | iOS 앱 | iPhone/iPad | 이동 중 세션 이어받기 |
| **웹 계열** | Chat (claude.ai) | 브라우저 | 가벼운 대화 |
| | Projects | claude.ai 프로젝트 | 배경 자료 누적 |
| | **Code on the Web** 🆕 | claude.ai/code | 로컬 없이 GitHub 레포에서 바로 작업, `--teleport`로 로컬 이동 |
| **협업/외부** | Cowork | Claude Desktop의 Cowork 탭 | 비동기 위임, Scheduled Tasks |
| | Slack 1급 연결 🆕 | 팀 Slack 워크스페이스 | `@Claude` 멘션으로 PR 받기 |
| | Chrome 확장 (beta) 🆕 | Chrome 브라우저 | 라이브 웹앱 디버깅 |
| **입력 방식** | Voice dictation 🆕 | CLI/Desktop 내 마이크 | 음성으로 프롬프트 입력 |

### 4경로 → 12 Surface 매핑

```
Chat        → claude.ai
Projects    → claude.ai 프로젝트
Cowork      → Desktop 앱 Cowork 탭, Slack 연결
Claude Code → CLI, Desktop, Web, VS Code, JetBrains, Chrome, iOS
              (+ Voice dictation은 입력 보조)
```

**핵심 원칙 2가지:**
1. **세션은 하나** — `claude --teleport`, `/desktop`, `claude --remote-control`로 Surface 간 이동 가능. Ch.24에서 상세
2. **설정은 공유** — CLAUDE.md, MCP, Skills, Hooks가 Surface에 관계없이 동일 적용

> 이 책은 **Claude Code CLI + Cowork** 중심으로 전체 체계를 배운 후, Part 7 Ch.25(Surface 넘나들기)에서 나머지 Surface를 한 번에 훑는다. 다른 Surface는 "이름만 기억하고 넘어간다" 정도로 충분하다.

## 4경로 비교표

| | Chat | Projects | Cowork | Claude Code |
|---|------|----------|--------|-------------|
| **핵심** | 질문과 답 | 배경 누적 | 문서 위임 | 코드와 자동화 |
| **비유** | 카페 대화 | 사무실 책상 | 비서에게 맡기기 | 옆자리 개발자 |
| **내가 있어야?** | 예 | 예 | **아니오** | 예 |
| **결과물 형태** | 채팅 텍스트 | 채팅 + 파일 | **정리된 문서** | **파일 직접 수정** |
| **반복 자동화** | 불가 | 불가 | **가능** | **가능** |
| **파일 접근** | 업로드만 | 업로드만 | 폴더 구조 | **로컬 전체** |
| **외부 도구** | 제한적 | 제한적 | Plugin/Connector | **MCP/Plugin/Hook** |

**어디서 시작하고, 무엇을 받는가:**

| 경로 | 시작점 | 받는 것 |
|------|--------|---------|
| Chat | claude.ai 대화창 | 채팅 텍스트 (복사해서 쓰기) |
| Projects | claude.ai 프로젝트 폴더 | 배경 누적된 대화 + 첨부 파일 |
| Cowork | 브리프 파일 → 맡기기 | 정리된 문서 (CLAUDE-OUTPUTS/) |
| Claude Code | 터미널에서 `claude` | 직접 수정된 파일, 커밋, 테스트 결과 |

**핵심 판단 기준:**
- "빠르게 물어보고 싶다" → **Chat**
- "같은 맥락을 계속 쓰고 싶다" → **Projects**
- "맡기고 결과만 받고 싶다" → **Cowork**
- "직접 파일을 만들고 코드를 돌리고 싶다" → **Claude Code**

## LLM OS — 클로드는 운영체제에 가깝다

AI 업계에서 자주 나오는 개념이 있다. **LLM OS**. 안드레이 카파시(Andrej Karpathy)가 2023년에 처음 언급한 이 개념은, LLM을 "자연어로 프로그래밍하는 운영체제"로 보는 시각이다. 2025년에 이르러 이 비유는 현실이 되고 있다.

클로드도 마찬가지다. 단순한 소프트웨어가 아니라, 4개 층으로 구성된 시스템이다.

### 인터페이스 층
사람은 자연어, 대화, 이미지, 음성으로 작업을 건넨다. 마우스와 키보드 대신 **말과 글**이 입력 장치다.

### 상태 층
CLAUDE.md, memory, brief, decision log, handoff 같은 파일들이 **영구적인 맥락**을 형성한다. 대화가 끊겨도 이 파일들이 남아 있으면 클로드는 "어디까지 했는지" 알 수 있다.

### 실행 층
Bash, Read/Edit/Write, MCP 도구, 브라우저, 커넥터, scheduled task — 클로드가 **실제로 행동하는** 도구들이다.

### 통제 층
approval, hook, sandbox, review, replay — 클로드가 **잘못된 방향으로 가지 않도록** 위험 반경을 줄이는 장치들이다.

이 4개 층이 함께 작동하면, 클로드는 "대화하는 AI"가 아니라 **"작업을 수행하는 시스템"**이 된다. 이 책은 이 4개 층을 하나씩 이해하는 과정이다.

## 이 책은 어디에 집중하는가

이 책은 **Claude Code + Cowork** 중심이다.

**이유:**
1. Claude Code가 클로드의 모든 기능(Skills, Hooks, MCP, SubAgent)을 활용할 수 있는 가장 강력한 경로
2. Cowork이 "나 없이도 일하는 클로드"를 가능하게 하는 유일한 경로
3. 이 두 경로를 이해하면, Chat이나 Projects는 자연스럽게 쓸 수 있다 (반대는 어렵다)

Chat부터 시작해도 물론 괜찮다. 하지만 Chat만 쓰면 클로드의 20%밖에 활용하지 못한다. 이 책은 나머지 80%를 여는 열쇠다.

## 제품 선택에서 가장 흔한 오해 3가지

### 오해 1: "도구가 같으니 어디서 써도 비슷하겠지"
아니다. 같은 모델이어도 Chat, Cowork, Claude Code에서 체감이 크게 다르다. Chat에서는 "좋은 답"을 기대하지만, Claude Code에서는 "실행된 결과"를 기대한다.

### 오해 2: "Cowork는 쉬운 버전, Claude Code는 어려운 버전이다"
아니다. 난이도보다 **위임 방식과 통제 방식**이 다른 것이다. Cowork는 비동기 위임, Claude Code는 동기 협업. 어떤 일이냐에 따라 적합한 경로가 다르다.

### 오해 3: "처음부터 모든 경로를 다 배워야 제대로 쓸 수 있다"
아니다. 한 경로에서 작은 성공을 만들고 넓혀가는 게 훨씬 빠르다. 이 책은 Claude Code부터 시작해서 Cowork으로 확장하는 순서를 따른다.
