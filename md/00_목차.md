## Part 2: 클로드의 기억과 설정

### Ch.5 클로드의 기억
- 왜 매번 같은 말을 반복하게 되는가
- **Memory 시스템**: 글로벌 vs 프로젝트 (user/feedback/project/reference 타입)
- **CLAUDE.md** — 프로젝트 운영 계약서
- **.claude/ 폴더 전체 구조**
- **초보자 필수 파일 8종**
- 🐍 문자열과 f-string — 프롬프트도 결국 텍스트 조합

### Ch.6 클로드의 설정
- **settings.json** 완전 해부
- **설정 우선순위**: Managed > Command Line > Local > Project > User
- **권한 모드 5종 비교**:
  | 모드 | 목적 | 장점 | 주의 |
  |------|------|------|------|
  | default | 기본 승인 포함 | 안전 | 승인 피로 |
  | plan | 계획 우선 | 실수 방지 | 속도 느림 |
  | acceptEdits | 편집 수락 활성화 | 빠름 | 파일 위험 |
  | auto | 분류기가 권한 자동 판단 | 승인 피로↓, 위험 차단 | preview 단계 |
  | bypassPermissions | 권한 건너뛰기 | 최고 속도 | 위험 |
- **allowlist vs deny**
- **scope**: project vs organization
- 🐍 조건문(if/else) — 권한 관리도 결국 조건 분기

### Ch.7 클로드의 규칙
- **CLAUDE.md vs rules/ vs settings.json**
- **path-specific rules 실전**
- 규칙을 파일별로 쪼개야 하는 이유
- **파일 네이밍 규칙**: `YYYY-MM-DD_topic_type.md`

### Ch.8 클로드의 습관 — Hooks
- Hook이란: "이런 일이 생기면 이렇게 해"
- **Hook 4가지 타입**: command / prompt-based / agent-based / HTTP
- **hooks 구조** (settings.json 내): PreToolUse, PostToolUse, matcher
- **Conditional `if` hooks** — 조건 만족할 때만 발화
- 실전: rm 실행 전 확인 훅
- 실전: Edit 후 자동 lint 훅
- 🐍 함수(def) — Hook도 결국 "이벤트 → 함수 호출"
- Hook과 allowlist의 관계

### Ch.13 도구 연결하기 — MCP
- MCP란, 클라이언트-서버 모델
- **transport 3종**: stdio(로컬) / HTTP(원격, 권장) / SSE(deprecated)
- **OAuth 2.0 원격 인증** — `--client-id`, `--callback-port`, `--oauth-scope`
- MCP 서버 설치하고 연결하기 (`claude mcp add`, `add-json`, `remove`, `serve`)
- 실전: 파일시스템 / Playwright MCP
- **3종 primitive**: Tools / Resources / Prompts
- **동적 업데이트**: `list_changed`, 재연결 backoff
- **Elicitation**: form / text / confirm 3모드
- **`claude mcp reset-project-choices`** — 프로젝트 scope 초기화
- **MCP는 최소 연결, 실제 필요한 것만**

### Ch.14 Plugins & Connectors
