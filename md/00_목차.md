## Part 1: 처음 만나는 클로드

### Ch.1 클로드가 뭔데?
- AI가 아니라 "작업 파트너"
- 클로드의 4가지 얼굴: Chat, Projects, Cowork, Claude Code
- 이 책은 터미널(Claude Code) 중심입니다

### Ch.2 터미널에서 첫 대화
- Claude Code 설치하기
- 첫 대화: "안녕, 클로드"
- 기본 명령어 5개
- 🐍 파이썬 맛보기: `print("안녕, 클로드")`

### Ch.3 클로드와 잘 대화하기
- 좋은 질문 vs 나쁜 질문
- 컨텍스트가 전부다
- Slash Commands: /help, /clear, /compact, /model, /cost
- 🐍 변수와 자료형 — 클로드도 결국 "입력 → 처리 → 출력"

### Ch.4 에러는 클로드의 피드백이다
- 에러 메시지 읽는 법
- 에러를 클로드에게 붙여넣는 습관
- "안 돼요"가 아니라 "이렇게 안 됐어요"로 질문하기

## Part 2: 클로드가 기억하는 법

### Ch.5 클로드의 기억
- 왜 매번 같은 말을 반복하게 되는가
- Memory 시스템: 글로벌 vs 프로젝트
- CLAUDE.md — 클로드에게 보내는 영구 편지
- .claude/ 폴더 전체 구조
- 🐍 문자열과 f-string — 프롬프트도 결국 텍스트 조합

### Ch.6 클로드의 습관
- Hook이란: "이런 일이 생기면 이렇게 해"
- PreToolUse, PostToolUse 실전
- 안전망 만들기: rm 실행 전 확인
- 🐍 함수(def) — Hook도 결국 "이벤트 → 함수 호출"

### Ch.7 클로드의 설정
- settings.json: 권한, 환경변수, 모델 고정
- 모델 선택 전략: Opus vs Sonnet vs Haiku
- Extended Thinking: 클로드에게 생각할 시간 주기
- /model, /cost — 비용 감각 기르기

## Part 3: 클로드에게 능력을 주다

### Ch.8 기술 가르치기 — Skills
- Skill이란: 폴더 하나가 지식 패키지
- SKILL.md 구조와 작성법
- 나만의 첫 Skill 만들기
- 🐍 반복문(for/while) — 반복 작업을 자동화하는 원리

### Ch.9 도구 연결하기 — MCP
- MCP란: 클로드와 외부 세계를 잇는 다리
- MCP 서버 설치하고 연결하기
- 실전: 파일시스템 MCP로 로컬 파일 읽기
- 🐍 딕셔너리 — JSON 구조를 이해하면 MCP가 보인다

### Ch.10 비교와 선택
- Skills vs MCP vs Plugins: 같은 문제, 세 가지 해법
- 판단 기준: 언제 뭘 쓰는가
- 실습: 하나의 문제를 세 방식으로 풀어보기

## Part 4: 클로드에게 맡기다

### Ch.11 위임이라는 개념
- Cowork이란: "심부름 보내기"
- Chat vs Claude Code vs Cowork — 언제 뭘 쓰나
- 첫 위임: 간단한 작업 보내고 결과 받기

### Ch.12 결과를 받는 법 — Channels
- 채널이란: 결과물이 도착하는 곳
- 이메일, Slack 등 채널 설정
- 결과물 형식 지정

### Ch.13 자동화 — Scheduled Tasks
- 예약 작업: "매주 월요일 오전 9시에 해줘"
- Self-contained 프롬프트 작성법
- 실행 확인과 실패 대처
- 🐍 파일 I/O — 자동화의 기본은 읽고 쓰기

### Ch.14 외부에서 시작하기 — Remote Control
- 외부 트리거로 클로드 깨우기
- Claude Code와 Cowork의 연계
- 실전 파이프라인: 트리거 → 실행 → 채널 수신
