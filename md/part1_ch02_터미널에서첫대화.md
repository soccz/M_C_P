> **이 챕터를 마치면**: Claude Code가 설치되어 있고, 터미널에서 클로드와 첫 대화를 나눠본 상태. 기본 명령어 5개를 알고, 파이썬 코드를 한 줄 실행해본 경험이 생긴다.

## 배경: 터미널이 뭔데?

컴퓨터를 쓸 때 우리는 보통 마우스로 아이콘을 클릭한다. 파일을 열고, 폴더를 만들고, 프로그램을 실행한다. 이게 **GUI**(Graphical User Interface) — 그림으로 된 인터페이스다.

터미널은 같은 일을 **글자로** 하는 곳이다.

- 파일을 열고 싶으면 → 아이콘 더블클릭 대신 `open file.txt`
- 폴더를 만들고 싶으면 → 우클릭 → 새 폴더 대신 `mkdir my-folder`
- 프로그램을 실행하고 싶으면 → 아이콘 클릭 대신 `python my_code.py`

처음 보면 무섭게 생겼다. 까만 화면에 커서가 깜빡이고 있으니까. 하지만 실제로 하는 일은 마우스 클릭과 같다. 표현 방식만 다를 뿐이다.

**왜 터미널을 쓰는가?**

마우스로 100개 파일의 이름을 바꾸려면 100번 클릭해야 한다. 터미널에서는 한 줄이면 된다. 클로드도 마찬가지다. claude.ai 웹에서 "이 파일 수정해줘"라고 하면 결과를 복사해서 붙여넣어야 한다. Claude Code에서는 클로드가 **직접 파일을 수정한다**. 이 차이가 크다.

## 터미널 여는 법

### Mac
- `Cmd + Space`를 눌러서 Spotlight 열기
- "Terminal"이라고 입력하고 Enter
- 또는: 응용 프로그램 → 유틸리티 → 터미널

### Windows
- `Win + R`을 눌러서 실행 창 열기
- "powershell"이라고 입력하고 Enter
- 또는: 시작 메뉴에서 "Windows Terminal" 검색

### Linux
- `Ctrl + Alt + T` (대부분의 배포판)

터미널을 열면 이런 모양이 보일 거다:

```
soccz@my-computer:~$
```

이 `$` 뒤에 명령어를 입력한다. `$`는 "준비됐어, 뭐 할래?"라는 뜻이다.

## Claude Code 설치하기 (30초 안에)

### 사전 준비

Claude Code를 설치하려면 **Node.js**가 필요하다. Node.js는 자바스크립트를 실행하는 환경인데, 지금은 "Claude Code를 돌리기 위한 기반 프로그램"이라고만 알면 된다.

Node.js가 있는지 확인:
```bash
node --version
```

버전 번호(예: `v20.11.0`)가 나오면 이미 설치되어 있다. "command not found"가 나오면 설치가 필요하다.

**Node.js 설치:**
- https://nodejs.org 에서 LTS 버전 다운로드 → 설치
- 또는 Mac: `brew install node`
- 또는 Linux: `sudo apt install nodejs npm`

### Claude Code 설치 (1줄)

```bash
npm install -g @anthropic-ai/claude-code
```

이 명령어의 뜻:
- `npm` — Node.js의 패키지 관리자 (앱스토어 같은 것)
- `install` — 설치해라
- `-g` — 전체 시스템에서 쓸 수 있게 (global)
- `@anthropic-ai/claude-code` — 설치할 프로그램 이름

설치가 끝나면 확인:
```bash
claude --version
```

버전 번호가 나오면 성공이다.

### 첫 로그인

```bash
claude
```

처음 실행하면 로그인을 요구한다. claude.ai 계정으로 인증하면 된다. Pro 플랜 이상이 필요하다.

> **💭 생각의 흐름**
>
> **문제** — 설치하다가 에러가 날 수 있다
> **질문** — "에러가 나면 어떡하지?"
> **발견** — 에러 메시지를 그대로 클로드에게 물어보면 된다 (claude.ai Chat에서)
> **결과** — 에러를 두려워하지 않는 첫 번째 연습이 된다 → Ch.4에서 상세히 다룸

## 첫 대화: "안녕, 클로드"

설치가 끝났으면, 터미널에서 이렇게 입력한다:

```bash
claude
```

잠시 후 대화형 인터페이스가 열린다. 프롬프트가 나타나면:

```
안녕, 클로드. 나는 이 도구를 처음 써봐.
```

클로드가 인사와 함께 기본 사용법을 알려줄 거다.

### 몇 가지 더 해보기

```
지금 이 폴더에 어떤 파일이 있어?
```

클로드가 `ls` 명령어를 실행해서 파일 목록을 보여줄 거다. **클로드가 직접 터미널 명령어를 실행한다**는 걸 체감하는 순간이다.

```
hello.txt라는 파일을 만들어줘. 내용은 "안녕하세요, 첫 파일입니다."로.
```

클로드가 파일을 직접 생성한다. `cat hello.txt`로 확인해보자.

```
이 폴더에 있는 모든 .txt 파일을 목록으로 정리해줘.
```

이런 것도 된다. Chat에서는 "이렇게 하세요"라고 설명만 해줬을 텐데, Claude Code에서는 **직접 실행하고 결과를 보여준다**.

## 명령 체계 전체 지도

Claude Code는 단순한 대화창이 아니다. 다양한 방식으로 실행할 수 있다. 지금 다 외울 필요는 없다. "이런 것들이 있구나" 정도로 훑어보자.

### 기본: 대화형 세션

```bash
claude                    # 대화 시작
claude "질문 내용"         # 한 줄 질문 후 대화 계속
```

가장 많이 쓰는 방식. 터미널에서 클로드와 주고받으며 작업한다.

### 비대화형 출력

```bash
claude -p "이 파일 요약해줘"        # --print 모드: 결과만 출력하고 끝 (-p는 --print의 단축형)
claude -p --output-format json "분석해줘"  # JSON 형식으로 출력
```

스크립트나 자동화에서 클로드를 호출할 때 쓴다. 대화 없이 결과만 받는다.

### 최소 하네스 모드

```bash
claude --bare              # 최소 설정으로 실행
```

hook, auto-discovery, background prefetch 없이 가볍게 시작. 빠르지만 기능이 제한된다.

### 세션 이어받기

```bash
claude --continue          # 마지막 세션 이어서
claude --resume            # 특정 세션 선택해서 이어서
claude --resume --fork-session  # 특정 세션의 맥락으로 새 세션 시작
```

이건 Part 5(세션 운영)에서 자세히 다룬다. 지금은 "이어서 할 수 있구나" 정도로만.

### 작업 분리

```bash
claude --worktree          # Git 분리 작업 폴더에서 실행
claude --worktree --tmux   # worktree + tmux 세션 (--tmux는 --worktree 필수)
```

### 원격 실행

```bash
# SSH로 원격 서버에 접속한 뒤 claude를 직접 실행
ssh user@host "cd /project && claude"
```

### MCP / Plugin 관리

```bash
claude mcp serve           # MCP 서버 시작
claude mcp add-json        # MCP 설정 추가
claude plugin list         # 설치된 플러그인 목록
claude plugin install      # 플러그인 설치
```

### 인증

```bash
claude auth login          # 로그인
claude auth status         # 현재 상태 확인
```

### Slash Commands (대화 중 사용)

대화 중에 `/`로 시작하는 명령어를 쓸 수 있다:

| 명령어 | 하는 일 |
|--------|---------|
| `/help` | 도움말 |
| `/clear` | 현재 대화 비우고 새로 시작 |
| `/compact` | 컨텍스트 압축 (긴 대화를 줄임) |
| `/memory` | 메모리 파일 확인 |
| `/init` | CLAUDE.md 초안 생성 |
| `/cost` | 토큰과 비용 확인 |
| `/status` | 상태 확인 |
| `/model` | 모델 전환 (opus/sonnet/haiku) |
| `/permissions` | 권한 확인 |

**지금 당장 기억할 것 5개:**
1. `claude` — 시작
2. `/help` — 모르겠으면
3. `/clear` — 대화 리셋
4. `/cost` — 비용 확인
5. `/exit` — 끝내기

나머지는 필요할 때 다시 찾아보면 된다. 부록 B에 전체 치트시트가 있다.

## 🐍 파이썬 맛보기: 터미널에서 코드 실행하기

클로드와 대화하는 것 말고, **직접 코드를 실행하는 경험**도 해보자. 아주 짧게.

### 파이썬 확인

```bash
python3 --version
```

버전 번호가 나오면 OK. 안 나오면:
- Mac: `brew install python3`
- Linux: `sudo apt install python3`
- Windows: https://python.org 에서 다운로드

### 한 줄 실행

```bash
python3 -c "print('안녕, 클로드')"
```

터미널에 `안녕, 클로드`가 출력된다.

이게 뭘 한 거냐면:
- `python3` — 파이썬 인터프리터를 실행해라
- `-c` — 다음에 오는 코드를 바로 실행해라
- `"print('안녕, 클로드')"` — 화면에 '안녕, 클로드'를 출력하는 코드

### 파이썬 파일 만들기

텍스트 편집기(VSCode 등)를 열고 이 내용을 입력한다:

```python
# hello.py — 나의 첫 파이썬 파일
print("안녕, 클로드!")
print("나는 파이썬으로 인사하는 중이야.")
print("1 + 1 =", 1 + 1)
```

저장한 뒤 터미널에서:

```bash
python3 hello.py
```

결과:
```
안녕, 클로드!
나는 파이썬으로 인사하는 중이야.
1 + 1 = 2
```

### 클로드에게 시켜보기

Claude Code 대화에서 이렇게 말해보자:

```
hello.py를 만들어줘. "안녕, 클로드!"를 출력하는 파이썬 파일이야.
```

클로드가 파일을 직접 만들어준다. 그리고:

```
방금 만든 hello.py를 실행해봐.
```

클로드가 `python3 hello.py`를 실행하고 결과를 보여준다. **내가 코드를 쓰지 않아도, 클로드에게 시키면 된다**. 하지만 결과가 맞는지 확인하려면 코드를 읽을 줄은 알아야 한다. 그래서 파이썬 기초를 같이 배우는 거다.

### print()가 뭔데?

`print()`는 파이썬에서 가장 기본적인 함수다. 괄호 안에 넣은 것을 화면에 출력한다.

```python
print("문자열")       # 글자를 출력
print(42)             # 숫자를 출력
print(1 + 2)          # 계산 결과를 출력
print("답:", 3 * 4)   # 여러 개를 섞어서 출력
```

지금은 이것만 알면 된다. 다음 챕터에서 변수와 자료형을 배운다.

## 잘 안 된다면

| 증상 | 원인 | 해결 |
|------|------|------|
| `command not found: claude` | 설치 안 됨 또는 PATH 문제 | `npm install -g @anthropic-ai/claude-code` 재실행 |
| `command not found: node` | Node.js 미설치 | nodejs.org에서 설치 |
| 로그인 실패 | Pro 플랜 아님 | claude.ai에서 플랜 확인 |
| `permission denied` | 관리자 권한 필요 | `sudo npm install -g @anthropic-ai/claude-code` |
| 느리게 시작됨 | 첫 실행 시 초기화 | 정상. 두 번째부터 빨라짐 |

에러가 나면 **에러 메시지를 복사해서 claude.ai Chat에 붙여넣어보자**. "이 에러 어떻게 해결해?" 라고 물으면 대부분 해결된다. 이것도 클로드 활용이다.

## 왜 이렇게 동작하는가

Claude Code는 내부적으로 이렇게 동작한다:

1. 내가 자연어로 말한다 → "이 폴더에 뭐가 있어?"
2. 클로드가 의도를 파악한다 → "파일 목록을 보고 싶구나"
3. 적절한 도구를 선택한다 → `ls` 명령어를 실행하자
4. 도구를 실행하고 결과를 받는다 → 파일 목록 획득
5. 결과를 자연어로 정리해서 보여준다 → "이 폴더에는 다음 파일이 있습니다..."

이걸 **agentic loop(에이전트 루프)**라고 한다. "관찰 → 생각 → 행동 → 반복"의 과정이다. Part 5에서 자세히 다루지만, 지금은 "클로드가 도구를 직접 부른다"는 것만 기억하면 된다.

이 과정에서 클로드는 **내 허락을 구한다**. "이 파일을 수정해도 될까요?"라고 물어보는 순간이 있을 거다. 이게 **permission gate(권한 관문)**다. 안전 장치다. Ch.6에서 자세히 다룬다.
