# Claude Thinking (C.T.) 프로젝트 규칙

## 프로젝트 개요
- **제목**: Claude Thinking — 클로드의 생각을 따라가는 교재
- **목표**: 파이썬 입문 + 클로드 완벽 이해 가이드
- **대상**: 프로그래밍 경험 없는 사람
- **철학**: "너보다 너의 기능을 잘 아는 사람은 없다" — 클로드에게 클로드를 물어봄
- **톤**: 블로그 (캐주얼, 경험 중심, 2인칭)

## 출력 형식
- **MD**: 해설집/교과서 스타일 — 챕터별 깊이, 상세 설명
- **HTML**: 발표 자료 스타일 — 시각화, 인터랙티브, 움직이는 요소
- **HTML 컬러**: 클로드 오렌지(#E87B35) + 검정(#1a1a1a) 조합

## 파이썬 규칙
- 클래스/모듈 제외
- 기여도 높은 챕터에만 🐍 배치 (전 챕터 균등 배분 금지)
- 파이썬은 "보조 언어" — 클로드 이해를 위한 도구

## 구조 원칙
- "생각의 흐름" 박스: 문제 → 질문 → 발견 → 결과
- 각 챕터: 훅 문장 → 배경 → 개념 → 실습 → 원리 → 변형 → 연결
- 실습 비율 40% 이상
- 에러 케이스 의도적 포함
